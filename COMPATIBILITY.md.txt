# Ma Pao's Homeschool Tutor — Cross-Platform Compatibility Policy

## 1. General Principles

- **Capability detection, never user-agent sniffing.**  
  Use `'speechSynthesis' in window`, `navigator.userAgentData`, `'SpeechRecognition' in window`, etc.  
  UA strings are unreliable and break over browser updates.

- **Graceful degradation.**  
  If a feature is unavailable, fail silently or show a plain-text fallback — never throw an unhandled error.

- **Pointer events first, mouse/touch as fallback.**  
  Use `onclick` or `addEventListener('pointerup', ...)`. Do not rely on `touchstart` alone.

---

## 2. Text-to-Speech (TTS) — `speechSynthesis` API

### Feature detection
```js
if (!('speechSynthesis' in window)) {
  // TTS not available — hide Read button, do nothing
  return;
}
```

### Known platform quirks and mitigations

| Platform / Browser | Quirk | Mitigation in code |
|---|---|---|
| **Edge (Chromium) desktop & mobile** | `speak()` silently fails: `onstart` never fires, no `onerror`, `speaking` stays `false` | 800 ms watchdog after `speak()` — if still not speaking, `cancel()` + retry once. Detected via `navigator.userAgentData.brands` (no UA string). |
| **Windows / Chrome / Edge** | `speechSynthesis` freezes mid-utterance (paused flag, or `speaking=true` but silent) | `resumeHack` interval: resumes if `paused`; restarts chunk if stuck >14 s with `isReading===false`. |
| **Windows / Chrome / Edge** | Second `cancel()` in same tick kills queued speech | Single `cancel()` in `readAloud()` / `startReadAloud()`, 400 ms delay before `speak()`. No second `cancel()` in `_doReadAloud`. |
| **iOS Safari** | `speak()` must be called synchronously inside a user gesture | `startReadAloud()` calls `doSpeak()` immediately (no `setTimeout`) when `hasPending` is true. |
| **All browsers** | `onstart` may fire late (especially Edge) | 1 s fallback timer in `speakChunk()` sets button state even if `onstart` hasn't fired. |
| **Firefox** | `onboundary` not implemented (word highlighting skipped) | `onboundary` handler is present but word index advances only on event — no highlight on Firefox; acceptable degradation. |
| **Android Chrome** | Voices list empty until `onvoiceschanged` fires | `ensureVoices()` waits for the event before calling `_doReadAloud`. |

### Audio codec fallbacks
This app uses the browser `speechSynthesis` API (no audio files are served). Codec fallbacks are therefore N/A. If a future version adds pre-recorded audio, serve both **MP3** and **OGG** and use `<audio>` `canPlayType()` to select:
```js
const audio = new Audio();
const src = audio.canPlayType('audio/ogg') ? 'file.ogg' : 'file.mp3';
```

### `play()` promise handling (for any future `HTMLAudioElement` use)
```js
const p = audio.play();
if (p !== undefined) {
  p.catch(err => {
    // Autoplay blocked — show a "Tap to play" prompt
    console.warn('Autoplay blocked:', err);
  });
}
```

## 5. Test Cases & Pass/Fail Criteria

### TC-01 · Read button — basic playback
- **Action:** Click/tap ▶ Read on a page with AI response text.
- **Pass:** Audio plays within 1.5 s; ▶ Read hides, ⏸ Pause appears.
- **Fail:** No audio after 3 s, or button stays as ▶ Read.

### TC-02 · Edge silent-fail watchdog
- **Action:** Click ▶ Read in Edge (desktop or mobile).
- **Pass:** Audio starts within 1.6 s (800 ms watchdog + 200 ms retry + buffer).
- **Fail:** No audio after 3 s.

### TC-03 · Pause / Resume
- **Action:** Click ⏸ Pause mid-speech; click ▶ Resume.
- **Pass:** Audio pauses and restarts at same position; buttons update correctly.
- **Fail:** Pause has no effect, or Resume restarts from beginning.

### TC-04 · Word highlighting
- **Action:** Click ▶ Read on Chrome or Edge.
- **Pass:** Words highlight in sync with speech; auto-scrolls to current word.
- **Fail:** No highlighting, or highlighting is static/wrong word.
- **Note:** Firefox — highlighting not expected (onboundary not supported); acceptable.

### TC-05 · Stop during playback
- **Action:** Navigate away or click ✗ Stop mid-speech.
- **Pass:** Audio stops immediately; buttons reset; no ghost speech continues.
- **Fail:** Audio continues or buttons stuck in playing state.

### TC-06 · Re-read (second tap without new response)
- **Action:** Tap ▶ Read, let it finish, tap ▶ Read again.
- **Pass:** Plays again from beginning without error.
- **Fail:** Silent fail, error in console.

### TC-07 · TTS on iOS (gesture requirement)
- **Action:** Open app cold in Safari iOS; tap ▶ Read without using mic first.
- **Pass:** Audio plays immediately on first tap.
- **Fail:** Silent fail; audio plays only after mic is used.

### TC-08 · No `speechSynthesis` — graceful degradation
- **Action:** Simulate `window.speechSynthesis = undefined` via devtools.
- **Pass:** Read button either hidden or click does nothing; no JS error.
- **Fail:** Uncaught exception thrown.

### TC-09 · resumeHack — stuck speech recovery
- **Action:** On Windows/Chrome, let a long response play; observe console for hack firing.
- **Pass:** Speech continues without going silent; hack fires < once per session.
- **Fail:** Speech goes permanently silent mid-text with hack not recovering it.

### TC-10 · Race condition — rapid button clicks
- **Action:** Click ✓ Correct, ✗ Wrong, Hint, or Send buttons multiple times rapidly.
- **Pass:** Only one AI evaluation fires; buttons re-enable after response.
- **Fail:** Multiple API calls fire, or buttons stay disabled permanently.

**Commit message:**

```
fix(tts): fix 3 Edge bugs — .pending mask, gesture window, watchdog double-fire

Bug 1: Watchdog .pending guard prevented retries entirely.
  Edge keeps .pending=true even after silently dropping speak().
  Guard removed — watchdog now fires correctly.

Bug 2: 120 ms setTimeout in startReadAloud() exceeded Edge Mobile
  gesture window (~100 ms), causing silent autoplay-block on every
  fresh tap. Fresh-tap path (hasPending) now calls doSpeak() synchronously.

Bug 3: watchdogFired flag added to prevent both 800 ms and 1500 ms
  probes calling speakChunk() simultaneously when retry is needed.

No behaviour changes on Chrome, Firefox, or Safari.
All five async buttons remain managed together per race-condition fix.

Closes: Edge Read / Tap to Listen not working (desktop + mobile)
```

**PR description:**
> **Root cause (3 bugs):**
> 1. `if(window.speechSynthesis.pending) return` inside the watchdog. Edge leaves `.pending=true` after silently dropping an utterance, so the watchdog never retried.
> 2. `setTimeout(doSpeak, 120)` on fresh tap exceeded Edge Mobile's ~100 ms autoplay-gesture window.
> 3. Both watchdog probes (800 ms + 1500 ms) could call `speakChunk()` concurrently if both fired before the retry resolved.
>
> **Files changed:** `index.html` — 3 targeted edits in `speakChunk()` and `startReadAloud()`.
>
> **Test evidence needed:** Click ▶ Read in Edge desktop and mobile immediately after page load. Audio should start within 1.5 s. Console should show `[TTS Edge] onstart fired` or `[TTS Edge] silent-fail watchdog fired (attempt 1)` followed by audio.
